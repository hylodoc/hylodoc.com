Blogging for programmers

We're building Xr0 and enjoy blogging about it.

But existing static-site generators like Jekyll and Hugo are powerful but lack reader-interaction features such as subscriber emails, metrics and so on. There's nothing wrong with this — they're literally called static‐site generators — but we want more.

And although Substack offers great interactivity, but isn't built for programmers. It has poor support for code: no syntax highlighting and the inline code rendering is jarring. Plus, we want to manage our blog posts as text files with Git.

Enter Hylodoc: Substack for programmers 🥳

  • Markdown and git first: Generate sites from Git repos. Markdown files are pages and code looks beautiful. Simply push your changes to a configured branch to apply changes to your site.
  • Separation of form and content: No custom CSS — pick a theme and start writing.
  • Batteries included: Email lists, metrics, custom domains, one‐click subscriber export are built-in. Prefer plaintext emails? We got you covered.
  • Open source: Both the platform and SSG are open to the community.

🚀 Launch a Hylodoc in 5 minutes

  1. Create a new GitHub repository with a file called index.md.
  2. Log into hylodoc.com click "Create new blog," install the GitHub App, and generate your blog.
